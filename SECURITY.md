# Security Policy

## Supported Versions

We actively support and provide security patches for the following versions of PII Shield:

| Version | Supported |
| ------- | --------- |
| 1.0.x   | ✅ Yes    |
| < 1.0   | ❌ No     |

## Reporting a Vulnerability

We take the security of PII Shield seriously. If you find a vulnerability or potential security issue in this repository (or Chrome Extension context), please do **not** open a public issue. Instead, report it privately to the maintainer:

* **Contact:** Andrew Yeo
* **Email:** [andrew.yeo.mc@gmail.com](mailto:andrew.yeo.mc@gmail.com)

Please include as much detail as possible in your report, including:
- A description of the vulnerability.
- Steps to reproduce (proof of concept).
- Potential impact of the issue.

We will acknowledge receipt of your report within 48 hours and coordinate a mitigation timeline.

## Disclosure Process

1. Once a vulnerability report is received, the maintainer will verify the issue.
2. A private draft security advisory will be created if necessary.
3. Patches will be developed and verified in a private staging workspace.
4. We will coordinate a release date for the security patch.
5. The fix will be committed, a new release published, and the advisory made public.
